# 10 · Quality requirements

<!-- arc42 section 10: the quality goals features are reviewed against.
     Scenarios use the form: stimulus → expected response. -->

| # | Quality | Scenario (stimulus → response) |
| --- | --- | --- |
| Q1 | Reliability | Peak concurrent use during the event → core attendee flows keep working without errors |
| Q2 | Attendee experience | An attendee opens the app during the event → schedule, sessions, and key actions work without organizer help |
| Q3 | Partner experience | A partner uses the partner platform → presence and participant contact work without organizer help |
| Q4 | Maintainability | A new volunteer with basics in the shared stack language joins → productive on a scoped issue within one week using repo docs alone |
| Q5 | Privacy | A participant asks what data the platform holds → answerable, exportable, deletable per GDPR |
| Q6 | Authorization | A partner queries platform data → sees only what their persona and ownership allow, never another partner's or the full attendee list |
